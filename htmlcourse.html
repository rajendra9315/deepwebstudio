MODULE 1: How the Web Works (Deep Understanding)

1.1 What is the World Wide Web?

The World Wide Web (WWW) is a system of interconnected documents written mainly in HTML. These documents are stored on servers and accessed through browsers using the internet.

HTML is the foundation layer. Everything else (CSS, JavaScript, frameworks) is built on top of it.

1.2 Client–Server Architecture

The web works on a client–server model. 

Client (Browser)
     ↓ HTTP Request
Server
     ↓ HTTP Response
Client renders page

Client Responsibilities

  • Send requests (URLs)
  • Receive responses
  • Render HTML
  • Execute CSS & JavaScript

Server Responsibilities

  • Store website files
  • Handle requests
  • Send correct responses
HTML ALWAYS comes first. Without HTML, nothing appears on screen.

1.3 What Happens When You Enter a URL?

1. User types URL
2. DNS lookup happens
3. Browser finds server IP
4. Server receives request
5. Server sends HTML
6. Browser builds DOM
7. CSS applied
8. JavaScript executed
9. Page rendered

1.4 Browser Rendering Pipeline (ADVANCED)

HTML → DOM
CSS  → CSSOM
DOM + CSSOM → Render Tree
Render Tree → Layout
Layout → Paint → Composite
Bad HTML structure = slower rendering + poor SEO.

MODULE 2: HTML Fundamentals (Strong Base)

2.1 What is HTML?

HTML stands for HyperText Markup Language. It is NOT a programming language.

HTML describes:

  • Structure
  • Meaning
  • Content hierarchy

2.2 HTML Document Structure (MANDATORY)

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Website</title>
  </head>

  <body>
    Content visible to users
  </body>
</html>

Why Each Part Matters

  • DOCTYPE → enables modern HTML
  • html → root element
  • head → metadata & SEO
  • body → visible content
Create a basic HTML page that displays:
  • Your name
  • Your profession
  • Your city

2.3 HTML Is NOT Visual Design

HTML does NOT control:

  • Colors
  • Fonts
  • Layout

That is CSS’s job. HTML is about structure and meaning.

MODULE 3: Tags, Elements & Attributes (Deep)

3.1 What is a Tag?

A tag is a keyword enclosed in angle brackets. 

<p>Paragraph</p>

3.2 What is an Element?

An element includes:

  • Opening tag
  • Content
  • Closing tag
<h1>This is an element</h1>

3.3 What is an Attribute?

Attributes provide extra information about an element. 

<a href="https://example.com" target="_blank">
  Visit Example
</a>

Common Global Attributes

  • id
  • class
  • title
  • style (avoid inline)

3.4 Boolean Attributes (ADVANCED)

<input type="checkbox" checked>
<input type="text" required>

Boolean attributes do not need values.

3.5 Nesting Rules (IMPORTANT)

<p>
  This is correct nesting.
</p>
Incorrect nesting breaks layout, SEO, and accessibility.
Inspect any website and identify:
  • Tags used correctly
  • Attributes misused

MODULE 4: Text Content & Typography (Deep + Professional)

4.1 Why Text Matters in HTML

Text is the most important content on the web. Search engines, screen readers, and users all depend on well-structured text.

Advanced developers treat text as semantic data, not decoration.

4.2 Headings (h1–h6) — VERY IMPORTANT

Headings define the document hierarchy. They are NOT for styling — they are for structure. 

<h1>Main Page Title</h1>
<h2>Section Title</h2>
<h3>Sub-section</h3>

Rules Every Professional Follows

  • Only ONE <h1> per page
  • Never skip heading levels
  • Headings must describe content
Using headings only to make text bigger is a beginner mistake.

4.3 Paragraphs & Line Breaks

<p>This is a paragraph.</p>
<br>
<p>This is another paragraph.</p>

Use:

  • <p> for content blocks
  • <br> ONLY for line breaks (addresses, poems)

4.4 Text Emphasis & Meaning

<strong>Important information</strong>
<em>Emphasized text</em>

Correct Meaning

  • <strong> → importance
  • <em> → emphasis
Screen readers announce <strong> and <em> differently.

4.5 Quotes & Citations

<blockquote>
  Learning HTML deeply makes everything easier later.
</blockquote>

<q>Short inline quote</q>

4.6 Lists for Text Structure

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

When to Use Lists

  • Steps
  • Features
  • Navigation
Rewrite a long paragraph into:
  • Proper headings
  • Short paragraphs
  • Lists where applicable

MODULE 5: Links & Navigation (Advanced & SEO-Safe)

5.1 What is a Link?

Links connect documents. Without links, the web would not exist. 

<a href="https://example.com">Visit Example</a>

5.2 Types of Links

External Links

<a href="https://google.com" target="_blank" rel="noopener">
  Google
</a>

Internal Links

<a href="/about.html">About Us</a>

Anchor (Page) Links

<a href="#contact">Go to Contact</a>

<section id="contact">
  Contact Content
</section>

5.3 Link Attributes (PRO LEVEL)

  • href → destination
  • target → where to open
  • rel → security & SEO
<a href="https://site.com" rel="nofollow noopener">
  External Resource
</a>
rel="noopener" prevents security vulnerabilities.

5.4 Navigation Menus (Semantic)

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

5.5 Accessibility in Navigation

  • Clear link text
  • No "click here"
  • Logical order
Bad navigation = users leave + SEO penalty.

5.6 Advanced Navigation Thinking

Professional developers think:

  • Is this link useful?
  • Is it accessible?
  • Does it help SEO?
Create a navigation menu for:
  • Home
  • Services
  • Blog
  • Contact
Use only semantic HTML.

MODULE 6: Images, Video & Audio (Deep + Performance)

6.1 Why Media Matters

Media elements enhance user experience, but if misused they destroy performance. Advanced developers balance quality, accessibility, and speed.

Bad media handling is one of the biggest reasons websites are slow.

6.2 Images – Core Concepts

<img src="photo.jpg" alt="Professional profile photo">

Critical Image Attributes

  • src → image path
  • alt → accessibility + SEO
  • width / height → layout stability
Missing alt text is an accessibility failure.

6.3 Responsive Images (ADVANCED)

<img 
  src="image-small.jpg"
  srcset="image-small.jpg 480w,
          image-medium.jpg 768w,
          image-large.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 50vw"
  alt="Responsive example">

The browser chooses the best image based on screen size and resolution.

6.4 Lazy Loading (PERFORMANCE)

<img src="gallery.jpg" loading="lazy" alt="Gallery image">
Lazy loading delays image loading until needed → faster page load.

6.5 Video Elements

<video controls width="600">
  <source src="movie.mp4" type="video/mp4">
  Your browser does not support video.
</video>

Video Best Practices

  • Use compressed formats
  • Provide controls
  • Avoid autoplay with sound

6.6 Audio Elements

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
</audio>

6.7 Media Accessibility

  • Alt text for images
  • Captions for video
  • Transcripts for audio
Create a media gallery page with:
  • Responsive images
  • One video
  • One audio file

MODULE 7: Lists & Tables (Advanced & Real-World)

7.1 Lists – More Than Bullets

Lists represent structured information. They are used heavily by screen readers and search engines.

7.2 Unordered Lists

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

7.3 Ordered Lists

<ol>
  <li>Install tools</li>
  <li>Write HTML</li>
  <li>Test in browser</li>
</ol>

7.4 Nested Lists (ADVANCED)

<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
</ul>

7.5 Tables – When to Use

Tables should be used ONLY for tabular data, never for layout.

7.6 Table Structure (MANDATORY)

<table>
  <caption>Student Scores</caption>
  <thead>
    <tr>
      <th>Name</th>
      <th>Score</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Raj</td>
      <td>95</td>
    </tr>
  </tbody>
</table>

7.7 Accessibility in Tables (PRO)

  • Use <th> correctly
  • Use scope attributes
  • Add captions
<th scope="col">Name</th>
<th scope="row">Raj</th>
Tables without headers are unusable for screen readers.
Build a table for:
  • Products
  • Prices
  • Availability
Follow accessibility rules.

MODULE 8: Forms (Complete Guide – Beginner to Advanced)

8.1 Why Forms Are Critical

Forms are the primary way users communicate with websites. Every login, signup, search, payment, and contact system starts with HTML forms.

A weak form = broken user experience + lost data.

8.2 Basic Form Structure

<form action="/submit" method="post">
  <label>Name</label>
  <input type="text">
</form>

Key Attributes

  • action → where data is sent
  • method → GET or POST

8.3 All Common Input Types (COMPLETE)

<input type="text">
<input type="email">
<input type="password">
<input type="number">
<input type="tel">
<input type="url">
<input type="date">
<input type="time">
<input type="color">
<input type="range">
<input type="file">
<input type="checkbox">
<input type="radio">
<input type="hidden">

8.4 Labels & Accessibility (MANDATORY)

<label for="email">Email</label>
<input id="email" type="email">
Forms without labels are inaccessible and unprofessional.

8.5 Required, Pattern & Validation

<input type="email" required>
<input type="text" pattern="[A-Za-z]{3,}">

Built-in Validation Benefits

  • No JavaScript needed
  • Consistent browser behavior
  • Better UX

8.6 Select, Textarea & Datalist

<select>
  <option>India</option>
  <option>USA</option>
</select>

<textarea rows="4"></textarea>

<input list="cities">
<datalist id="cities">
  <option value="Delhi">
  <option value="Mumbai">
</datalist>

8.7 Real-World Form Example (Contact)

<form>
  <label>Name</label>
  <input type="text" required>

  <label>Email</label>
  <input type="email" required>

  <label>Message</label>
  <textarea required></textarea>

  <button type="submit">Send</button>
</form>
Create a registration form with:
  • Name

MODULE 10: Accessibility (WCAG – Beginner to Advanced)

10.1 What is Web Accessibility?

Web accessibility ensures that websites can be used by everyone, including people with visual, auditory, motor, and cognitive disabilities.

Accessibility is not optional. It is a professional, legal, and ethical requirement.

10.2 WCAG Principles (INTERVIEW LEVEL)

POUR
Perceivable
Operable
Understandable
Robust

Examples

  • Text alternatives for images
  • Keyboard navigation
  • Readable content
  • Compatible with assistive tech

10.3 ARIA Basics (WHEN NEEDED)

<button aria-label="Close dialog">X</button>
Do NOT use ARIA when native HTML elements already exist.

10.4 Keyboard Navigation

  • Tab order must be logical
  • Focusable elements must be visible
  • No keyboard traps

10.5 Accessibility Checklist

  • Alt text on images
  • Labels on inputs
  • Semantic landmarks
  • Contrast readability

MODULE 13: HTML Security Basics

13.1 What HTML Can and Cannot Secure

HTML alone cannot secure a website, but poor HTML can introduce vulnerabilities.

13.2 Common HTML-Level Risks

  • Unescaped user input
  • Improper form handling
  • Exposed sensitive data

13.3 Basic Security Best Practices

  • Never trust client input
  • Use proper input types
  • Avoid inline JavaScript

13.4 HTML & XSS Awareness

Cross-Site Scripting (XSS) often starts with poor handling of user input.

Security is primarily backend-driven, but HTML is the first defense layer.
Review a form and identify:
  • Inputs that need validation
  • Potential misuse
